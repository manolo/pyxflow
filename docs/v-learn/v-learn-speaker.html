The Thin Client Secure by Architecture

Zero Attack Surface

No REST APIs to exploit. No GraphQL endpoints. The protocol is opaque.

Minimal Bandwidth

Only UI diffs travel the wire.
No full-page reloads.
No heavy HTML/JS fragments to download.

Server-side State

All validation, all business logic, all data access stay on the server.
Nothing sensitive in the browser.

Vaadin uses a thin client.
The browser just renders the UI.
The server has everything. State, business logic, components.
→ This gives you 3 things for free.
→ First: zero attack surface. No REST APIs to hack.
→ Second: minimal bandwidth. Only UI diffs travel the wire.
→ Third: server-side state. Nothing sensitive in the browser.

        button = Button("Say hello", lambda e:

7 lines. A complete interactive web view.

grid.set_data_provider(self.fetch_page)

    return db.query(offset, limit), db.count()

Lazy loading, server-side sorting, single callback.

Grid is the most complex component. Thousands of rows loaded on demand. Sorting, column reordering, selection — all handled by one callback. No pagination widgets, no state management libraries, no infinite scroll JavaScript.

Real-time Push


@AppShell
@Push
class App: pass

async def _tick(self):
    while self._running:
        await asyncio.sleep(1)
        self._elapsed += 1
        ui.access(self._update_display)

Server pushes UI updates via WebSocket. No polling.

But we can also implement enterprise features like Push for real-time updates.
Just by adding the Push annotation.
And, Then use ui.access to update the browser from a background task.

→ We use here the standard python asyncIO. NO special libraries.

Three Separations

Thinking / Doing

Humans write specs (the "what"). AI generates code (the "how"). Each plays to their strength.

Strategy / Tactics

Specs define architecture decisions. Code is a tactical detail that can be regenerated.

Patterns / Projects

Specs are reusable across languages and projects. Code is a single instance.

Three key separations in SDD. Thinking vs Doing: humans are good at understanding systems, AI is good at generating code. Strategy vs Tactics: specs are durable, code is disposable. Patterns vs Projects: the same specs could generate a Rust port tomorrow.

Enterprise Patterns


binder = Binder(Person)
binder.for_field(self.name).bind("name")
binder.for_field(self.email) \
    .with_validator(EmailValidator()) \
    .bind("email")

@ClientCallable
def save(self):
    if binder.write_bean(self.person):
        service.save(self.person)

Binder + Validation + @ClientCallable — the enterprise CRUD pattern.

The most common enterprise pattern: Master-Detail. Grid on the left, form on the right. Binder does two-way data binding with validation. @ClientCallable lets the browser call server methods directly. Same patterns Java developers know, now in Python.
