The Architecture

The browser is a thin client. All logic runs on the server.

This is the essence of Vaadin, unchanged for 25 years. Server maintains a state tree of UI nodes. Changes are sent as diffs (UIDL protocol). The browser just renders — no business logic, no data access, no API endpoints exposed. The same protocol that powers Java Vaadin Flow now runs in Python.

Secure by Architecture

Zero Attack Surface

No REST APIs to exploit. No GraphQL endpoints. The WebSocket protocol is opaque and binary.

One Team, One Language

Full-stack Python. No JS specialists, no API contracts, no serialization layer.

Server-side State

All validation, all business logic, all data access — stays on the server. Nothing sensitive in the browser.

Three pillars from the Vaadin philosophy. Security: there is literally nothing to attack on the client. Team: one language means one team, faster iteration. State: no state sync bugs, no stale caches, no optimistic updates gone wrong.

Real-time Push


@AppShell
@Push
class App: pass

async def _tick(self):
    while self._running:
        await asyncio.sleep(1)
        self._elapsed += 1
        ui.access(self._update_display)

Server pushes UI updates via WebSocket. No polling.

@Push enables WebSocket. ui.access() safely modifies UI from background tasks. Standard asyncio — no special framework, no Socket.IO, no custom protocols.
