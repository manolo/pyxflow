# PyFlow -- Package Name Candidates for PyPI

## Key Concepts

1. **Server-side rendering** -- All logic on the server (Python), browser just renders
2. **Thin client** -- Frontend is "dumb", just web components + JSON protocol
3. **Flow** -- State flows from server to client (state tree, UIDL changes)
4. **Pure Python** -- No JS required, everything in Python
5. **Components** -- 49 professional UI components (Grid, Dialog, etc.)
6. **Protocol/Wire** -- Communication via UIDL protocol, not traditional HTML
7. **Secure by design** -- State lives on the server, client has no direct access
8. **Real-time** -- WebSocket push, live updates
9. **Full-stack** -- Single language for everything
